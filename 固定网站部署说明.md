# 固定网站部署说明

## 推荐方案：Render

1. 将项目文件上传到一个 GitHub 仓库。
2. 登录 Render，创建 `New Web Service`，连接该仓库。
3. Runtime 选择 `Node`。
4. Start Command 填写 `node server.js`。
5. Health Check Path 填写 `/health`。
6. 创建完成后，Render 会提供固定的 `https://你的服务名.onrender.com` 地址。

顾客打开固定地址，只需输入客服提供的房间号即可加入。

## 正式营业建议

- 使用不容易猜到的房间号，例如订单号加随机字符。
- 配置 Render 持久磁盘，并将环境变量 `DATA_DIR` 设置为磁盘目录。
- 购买自己的域名后，可在 Render 中绑定域名。
- 当前版本知道房间号即可进入，不包含客服登录、房间密码或顾客身份验证。
